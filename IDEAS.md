# GitLens — New Ideas

## Marketing / Landing Website
- Feature showcase site (gitlens.dev) — screenshots, demo video, quick-start
- Live demo instance — one-click "try it" with a pre-populated demo GitHub account
- Documentation hub — install guides, architecture diagrams, FAQ, changelog
- Blog — posts on DORA metrics, multi-repo monitoring tips, DevOps insights

## New Product Features
- Notifications — Slack/Discord/email alerts for workflow failures, new releases, stale PRs
- Dashboard customization — pin repos, reorder cards, choose visible metrics per user
- Search/filter repos — crucial for users tracking 50+ repos
- Charts & trends — time-series graphs for DORA metrics over weeks/months
- Status badge API — embeddable SVG badges (like Shields.io) for READMEs
- Organization/team support — monitor entire orgs, multi-user dashboards
- GitHub App — replace OAuth app for granular permissions and auto-webhooks
- Dashboard export — JSON/CSV export of metrics, or a REST API for external consumption

## UI / UX Improvements
- Public landing page at `/` — instead of a bare login prompt, show a polished landing with feature highlights, screenshots, and a CTA to "Login with GitHub"
- Mobile responsiveness — optimize for phones/tablets
- Dark/light theme toggle
- Onboarding wizard — first-time user flow to import repos and set sync intervals
